🎨 III. How CSS Works with HTML

CSS does not contain content; it only contains instructions for how the HTML content should look. Below is exactly how they "talk" to each other.

1. Linkage

The <link> tag acts as a bridge, telling the HTML where to find its styles.

2. Selection

CSS uses a "Selector" (the tag name) to target exactly which part of the HTML to change.

3. Decoration

The "Rules" inside the curly braces define the visual changes (colors, sizes, etc.).

index.html

<head>
  <link rel="stylesheet" href="style.css"> // <-- Bridge
</head>
<body>
  <h1>Hello World</h1> // <-- The Target
</body>

style.css

h1 { // <-- The Selector (Finding the target)
  color: blue; // Changing text color
  font-size: 24px; // Formatting size
}

How they connect:

  • Linkage: The line <link rel="stylesheet" href="style.css"> in the HTML tells the browser: "Import the rules from style.css."
  • Selection: The h1 in the CSS acts as an arrow → pointing directly to the <h1> tags in the HTML.
  • Formatting: The CSS is formatting the visual properties of the content found inside that specific HTML tag.

🚀 IV. Planning Ahead for Project 3: Repository Standards

Your project must be a single, public GitHub repository. Professional organization is required to ensure your live site and documentation are easily accessible.

Mandatory Repository Files:

  • README.md (Root): The project's homepage. Must explain the purpose and link to both the live URL and the docs/ folder.
  • index.html (Root): The main homepage of your website.
  • about.html (Root): A second page, linked from your homepage.
  • style.css (Root): External stylesheet linking to both HTML pages.
  • docs/ folder: Contains the following Markdown files:
    • scope.md: Revised Project Scope Statement.
    • plan.md: TAME Risk Analysis & Task Schedule (PMUPM Ch 4).
    • retrospective.md: Final project reflection (PMUPM Ch 7).

Correct Repository Structure:

project-three-repo / main
  • 📂 docs/
  • 📄 scope.md
  • 📄 plan.md
  • 📄 retrospective.md
  • 📄 index.html
  • 📄 about.html
  • 📄 style.css
  • 📄 README.md

Deployment Instructions (GitHub Pages):

  1. Go to your repository Settings tab.
  2. Select Pages from the left-hand menu.
  3. Under Build/Deployment, set Source to Deploy from a branch.
  4. Select the main branch and /(root) folder.
  5. Click Save and wait 1-3 minutes for your live URL.

🧩 V. The Project 3 Puzzle

Think of Project 3 as a completed puzzle where every file type serves a distinct, vital role:

  • The Scope (Markdown): This is your "Picture on the Box." It defines the goals and boundaries.
  • The Structure (HTML): These are the "Puzzle Pieces." Semantic tags create the shapes (Header, Main, Footer).
  • The Presentation (CSS): This is the "Color and Detail." It transforms plain elements into a professional brand.
  • The Workflow (GitHub): This is the "Puzzle Board." The platform where you host, track, and publish.
⚠️ Mastery of Project 1 is essential: You cannot effectively style or manage a project without a solid structural foundation.
