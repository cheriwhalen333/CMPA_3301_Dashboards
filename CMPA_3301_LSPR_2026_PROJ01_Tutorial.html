Bridging Theory & Practice

Project 01 is your first opportunity to apply the project management principle of **"Scoping"** to a tangible technical task. By first defining the purpose, goals, and limitations of your personal webpage, you will experience how planning makes the technical work of writing HTML more focused and efficient.

✓ Learning Priorities

Planning > Coding

Success is defined by a well-structured document, not a complex design.

Semantic Structure

Learn the "tags" that provide context to search engines and screen readers.

The Technical Workflow

Master the pipeline from your local hard drive to the global web.

Workspace
Done?

Step 1: Open Your Environment

Sync your tools before writing code.

  1. Open **GitHub Desktop**.
  2. Select your **Project Repository** (e.g., `project-01`) from the "Current Repository" menu.
  3. Click **"Open in Visual Studio Code."** This ensures all files are saved in the correct place immediately.
Scoping
Done?

Step 2: Define the Scope (`scope.md`)

In VS Code, go to **File > New Text File**, then **Save As...** named exactly `scope.md`.

Include these headers and detailed responses:

# Project Purpose

Why are you building this site? (1-2 sentences)

# Description

What will the site contain visually and structurally?

# Desired Results

What does success look like for this project?

# Exclusions

What will you NOT include? (e.g., CSS, Javascript)

Coding
Done?

Step 3: Structure the HTML (`index.html`)

Create a new file named `index.html`. Type ! and press Enter for the skeleton.

Required Body Tags:

  • ☐ h1 - Your Full Name
  • ☐ h2 - Your Major / Interests
  • ☐ p (x2) - Career Goals paragraphs
  • ☐ ul/li - List of at least 3 hobbies
  • ☐ a - Hyperlink to your GitHub Profile
Cloud Sync
Done?

Step 4: Push to GitHub

Ensure your work is saved and published to the web.

  1. Go back to **GitHub Desktop**.
  2. Type a clear Summary (e.g., "Complete Project 01 Code").
  3. Click **Commit to main**.
  4. Click **Push origin** at the top bar.
Submission
Done?

Step 5: ZIP & Upload

Canvas requires a single file submission.

  1. In GitHub Desktop: **Repository > Show in Explorer/Finder**.
  2. Select BOTH `index.html` and `scope.md`.
  3. **Right-click > Compress** (Win: ZIP file / Mac: Compress).
  4. Verify the ZIP is under 1MB and upload to Canvas.
